## 三、UI/UX 问题详细分析

### 3.1 信息架构问题

| 问题 | 位置 | 影响 |
|------|------|------|
| Dashboard "开发顺序"是占位文案 | `DashboardPage.tsx:55-63` | 用户看到开发文档而非工具台 |

### 3.2 交互效率问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| 歌曲/分组只能上下按钮排序 | 🔴 高 | 歌单长了之后极慢 |
| 无行内编辑 | 🔴 高 | 修改歌曲只能删除重建 |
| 编辑器无"加法"能力 | 🟡 中 | 只能删不能加歌曲 |
| 无批量操作 | 🟡 中 | 曲库管理效率低 |
| 导出预览固定尺寸 | 🟡 中 | 用户看不到实际导出效果 |

### 3.3 移动端体验问题

| 问题 | 影响 |
|------|------|
| 导航栏 6 个链接在移动端拥挤 | `AppFrame.tsx` 在 900px 以下堆叠 |
| 编辑器控制面板和预览上下堆叠 | 移动端编辑时看不到预览效果 |
| 底部无固定操作栏 | 长页面操作按钮在顶部，操作不便 |

### 3.4 反馈与容错问题

| 问题 | 严重程度 | 说明 |

| 无自动保存 | 🟡 中 | 长时间编辑可能丢失 |
| 无操作成功 toast | 🟢 低 | 保存/发布/导出只有 inline-message |

## 四、代码架构问题

### 4.1 组件拆分不足

- **7 个页面文件平均 200+ 行**，全部是巨型组件
- `components/` 目录只有 3 个组件：`AppFrame`、`OnboardingPanel`、`ThemeSwitcher`
- 应提取：`SongCard`、`FilterBar`、`ConfirmDialog`、`Toast`、`EmptyState`、`PageLayout`

### 4.2 样式管理问题

- **单文件 CSS 1026 行**（`styles.css`），包含全部样式
- 无 CSS Modules 或按页面拆分
- 响应式断点只有 900px/720px，无 480px 精细断点

### 4.3 状态管理问题

- 无全局状态缓存，每个页面独立 `listSongs()`
- 编辑器状态无持久化，刷新页面丢失未保存编辑
- 无 `beforeunload` 提示

### 4.4 类型与数据"僵尸字段"

| 字段 | 存在位置 | 实际使用 | 状态 |
|------|---------|---------|------|
| `skinId` | `Playlist` 类型、`playlistBuilder.ts` | 全局主题覆盖，不生效 | 🧟 僵尸字段 |
| `thumbnail` | `Playlist` 类型 | 从未生成 | 🧟 僵尸字段 |
| `blur` | `BackgroundConfig` 类型 | 编辑器无 UI | 🧟 僵尸字段 |
| `sourceNote` | `LearningRequest` 类型 | 表单无输入 | 🧟 僵尸字段 |

## 七、关键决策记录

### 已确认的技术选择

- **状态管理**：继续用 hooks + local state，不引入 Redux/Zustand（项目规模适合）
- **拖拽库**：使用 `@dnd-kit/core` + `@dnd-kit/sortable`
- **样式方案**：保持 CSS 变量方案，逐步拆分文件（不引入 Tailwind）
- **数据验证**：备份恢复时手动校验，不引入 zod（减少依赖）

### 待确认的产品决策

1. 设置页面是否合并到 Dashboard 还是独立路由？
2. 歌曲编辑用行内编辑还是弹窗？
3. 缩略图在导出时生成还是打开编辑器时生成？
